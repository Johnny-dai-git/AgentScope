════════════════════════ Router (merged into llm-api) ═══════════════════════════
  Note: Current single-model setup (qwen2.5-0.5b). The original router's
        "multi-model scheduling" responsibility has been merged into llm-api,
        which forwards directly to vllm-worker. If BERT-style encoder models
        are introduced later (vLLM does not support them; a separate worker
        is required), the router can be split out again.

══════════════════════ Zero-Trust Network Policies (llm ns) ═════════════════════

   - Allow ingress-nginx → api-gateway-service
   - Allow llm-api Pods → vllm-worker-service
   - Default deny for all cross-namespace traffic
